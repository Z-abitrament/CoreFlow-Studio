# M17 寄存器列表库设计

## 目标

M17 将 Modbus 寄存器定义从单个 Device Profile 中分离出来，建立可复用、
可版本化的寄存器列表库。Device ID 只绑定一个明确的寄存器列表 ID 和版本，
变送器型号、测量管型号和设备型号不参与列表匹配。

本设计不引入“零点检测扩展包”。零点检测寄存器属于普通寄存器列表版本的
内容。只有完整基础列表经过确认后，才能发布包含新增零点检测寄存器的新版本；
不得用当前零点监视专用映射覆盖设备正在使用的完整列表。

## 数据对象

### 寄存器列表

每个列表版本包含：

- 稳定且不可变的 `register_map_id`。
- 独立的 `version`。
- 可修改的显示名称。
- `official`、`custom`、`legacy` 或 `template` 来源。
- 规范化内容的 SHA-256 校验值。
- 完整的寄存器定义 JSON。
- 创建和更新时间。

同一 `register_map_id` 可以有多个版本。同一 ID 和版本的内容不可原地替换；
不同内容必须创建新版本。客户端升级可安装新的官方列表版本，但不得自动改变
已绑定 Device ID 的版本。

### Device Profile

Device Profile 保存：

- 稳定 Device ID。
- 串口和 Modbus 地址配置。
- `register_map_id` 和 `register_map_version`。
- 当前有效寄存器列表快照，作为旧版本兼容和故障回退依据。
- 设备型号等可选元数据。

一个列表版本可以被多个 Device ID 复用。删除 Device Profile 不删除列表，
也不删除历史记录。

### 历史快照

运行、操作尝试和测试会话继续保存当次实际使用的完整寄存器列表快照。
更新列表库或 Device Profile 绑定不得回写已有历史记录。

## 旧数据迁移

SQLite schema v6 对每个仍内嵌寄存器表的旧 Device Profile：

1. 规范化 JSON 并计算 SHA-256。
2. 用内容校验值生成稳定的 `legacy-*` 列表 ID。
3. 相同内容只建立一个旧配置导入列表。
4. 将原 Device ID 绑定到该列表和原版本。
5. 保留 Profile 内嵌快照和所有历史快照。

迁移不根据空白或人工填写的设备型号猜测列表。

## 操作界面

Device Profile 对话框提供寄存器列表选择器：

- 选择已有列表版本时，表格载入该版本。
- `New List` 从当前表格建立新列表，要求列表 ID、显示名称和版本。
- 编辑已有自定义或旧配置列表时保存为新版本，只切换当前 Device ID。
- 官方列表不可原地修改；修改前必须建立自定义列表。
- 保存前可查看变量新增、删除和修改摘要。
- 连接期间禁止切换或修改列表。

## 客户端更新

安装包携带官方寄存器列表文件。应用启动时按 ID、版本和校验值导入列表库：

- 新 ID 或新版本自动加入列表库。
- 相同 ID、版本、校验值幂等跳过。
- 相同 ID、版本但内容不同视为发布冲突并拒绝覆盖。
- 已有 Device ID 继续使用原绑定版本，直到用户明确切换。

当前 `krohne_prj_main.json` 是从 `Krohne_prj` DSP 提交 `f0a1b39` 的
`mb_map_def.h` 和 `mb_map.c` 提取并校验的完整官方主列表，包含原有测量、参数、
通信、线圈、零点监测和 PI 驱动增益寄存器。它不是扩展包。源码提取器位于
`scripts/extract_krohne_register_map.py`；当 DSP 映射变化时，必须重新提取并发布
新的主列表版本，不能原地覆盖已有 ID/版本。

源码一致性检查命令：

```powershell
conda run -n coreflow-studio python scripts\extract_krohne_register_map.py `
  --dsp-root "<Krohne_prj路径>" `
  --output config\register_maps\krohne_prj_main.json `
  --check
```

## 设备身份与未来发现区

当前 DSP 没有固定的寄存器列表 ID/版本发现区，因此 M17 不根据设备型号自动
选择列表。当前连接顺序是先使用 Device Profile 已绑定的列表，再读取该列表
已经定义且编码明确的身份字段。

未来 DSP 应保留跨版本不变的只读发现区，至少提供协议标识、寄存器列表 ID、
寄存器列表版本和固件版本。客户端只有在 ID 和版本匹配时才可自动选择列表；
型号、测量管或传感器信息仅用于显示、记录和一致性告警。

## 安全与验证边界

- 列表管理和迁移在断开连接时完成，不产生 Modbus 请求。
- 本地测试使用 SQLite 临时库、寄存器映射夹具和 Qt 操作路径。
- 在线测试只验证选择后的读请求和历史快照；不包含任何设备写入。
- DSP 固定发现区和真实设备身份读取需在固件协议确定后单独验证。
